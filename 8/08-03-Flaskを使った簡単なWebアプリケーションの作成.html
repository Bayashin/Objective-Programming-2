WebアプリケーションフレームワークのFlaskを使ってみましょう

WebアプリケーションフレームワークFlaskの導入

Flaskは、PythonでWebアプリケーションを構築するためのライブラリです。

講義で説明する内容以外のことは、以下の公式サイトか、日本語翻訳版のどちらかを参照するようにしてください。

  • Welcome to Flask — Flask Documentation (2.0.x)
  • Flaskへようこそ — Flask Documentation (2.0.x)

Flaskを使った簡単なアプリケーションの作成

1. インストール

$ pyenv local 3.10.6 # これが行われているディレクトリで実行しましょう。
$ python -m pip install flask

# または

$ pip install flask

2. プロジェクトの作成

任意のディレクトリを作成し、そのディレクトリの中に以下のファイル、ディレクトリを作成してください。

[projectディレクトリ]
  ├[static]
  │  └ここには、CSSやJavaScriptなどの静的なファイルを格納します。今は空でOK
  ├[templates]
  │  └ここには、後述するJinja用のテンプレートHTMLを格納します。今は空でOK
  └main.py ⇠ プロジェクトのエントリポイント

flaskappというプロジェクトディレクトリを作成する場合は、以下のコマンドを使うと良いでしょう。 pyenvによるバージョンの切り替えを忘れずに行いましょう。

$ cd # ホームディレクトリに移動
$ cd work/oop2 # 作業ディレクトリに移動
$ pyenv local 3.10.6 # 必須
$ pip install flask # インストール済みであれば不要
$ mkdir flaskapp
$ cd flaskapp
$ mkdir static
$ mkdir templates
$ touch main.py
$ code .

3. main.pyのコーディング

以下のコードをmain.pyに書いてください。

from flask import Flask, request    # Flaskは必須、requestはリクエストパラメータを処理する場合に使用します。
app = Flask(__name__)


# http://localhost:8888/hello
@app.route('/hello')
def hello():
    return "<html><h1>Hello, Flask!!</h1></html>"


# GETパラメータの取得（クエリストリングより）
# http://localhost:8888/
@app.route('/')
def index():
    html = "<html><h3>index page</h3><ol>"

    # URL中のクエリストリングを処理(個別に取得したい場合は、request.args.get("hoge")が使えます)
    for key, value in request.args.items():
        html += "<li>{}: {}</li>".format(key, value)

    html += "</ol></html>"
    return html


if __name__ == "__main__":
    # debugモードが不要の場合は、debug=Trueを消してください
    app.run(host="localhost", port=8888, debug=True)

4. 実行してみる

プロジェクトディレクトリをターミナルで開き、以下のコマンドで実行します。

$ python main.py

動作確認はブラウザを開き、http://localhost:8888/hello、または、http://localhost:8888/にアクセスしてください。

また、index()関数で定義しているリクエストパスでは、リクエストに含まれるクエリストリングを解析して表示するプログラムも含まれています。以下のURLも試してみましょう。

  • http://localhost:8888/?name=HOGE&age=20
  • http://localhost:8888/?name=FUGA&age=120
  • http://localhost:8888/?hoge=HOGEHOGE&fuga=FUGAFUGA

5. プログラムの停止

実行しているプログラムは、[control] + [C]で停止させられます。

6. 機能を追加して動作確認をする

以下のコードをmain.pyに追加しましょう。 追加する場所は、if __name__ == "__main__":の前に差し込みます。


# GETパラメータの取得（REST APIに対応可能）
# http://localhost:8888/get/
# http://localhost:8888/get/<String>
@app.route('/get/')
@app.route('/get/<name>')
def get_param(name="no name"):  # nameパラメータが渡されなかった場合、「no name」が渡されます
    return "<html><h1>Hello, {}!!</h1></html>".format(name)


# GET,POSTどちらでもリクエストを受け付け、POSTの場合はリクエストボディを取得
# （コマンドラインでの動作確認は、$ curl -X POST -d "name=hoge" http://localhost:8888/post/ でできます。）
# http://localhost:8888/post/
@app.route('/post/', methods=["GET", "POST"])  # methods=["POST"]のみにすればGETメソッドでのリクエストはエラーにできる
def post_param():
    if request.method == 'POST':
        # name = request.form['name']
        name = request.form.get("name")
        return "<html><h1>Hello, {}!!</h1></html>".format(name)
    else:
        name = "no name"
        return "<html><h1>Hello, {}!!</h1></html>".format(name)

テンプレートエンジンを使用する

Flaskでは、Jinjaと呼ばれるテンプレートエンジンを採用しています。Flaskをインストールすると、内部的にはJinjaもインストール済みになります。

テンプレートエンジンのJinjaは、以下の公式サイトにてドキュメントを確認できます。

  • Jinja — Jinja Documentation (3.0.x)

1. テンプレートファイルの用意

以下のHTMLをtemplatesディレクトリ以下に、template1.htmlとして作成してください。

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>トップページ</title>
</head>
<body>
  <h1>Flask with Jinja Template</h1>
  {% if name %}
  <h3>Hello, {{ name }}!</h3>
  {% else %}
  <h3>Hello, Jinja!!</h3>
  {% endif %}
</body>
</html>

2. main.pyの修正

main.pyの1行目の以下の様にインポートに、rendar_templateを追加します。

from flask import Flask, request, render_template

続いて、以下の関数をmain.pyに追加してください。

# Jinjaを使ったテンプレートファイルのサンプル
# http://localhost:8888/template_sample/
# http://localhost:8888/template_sample/<String>
@app.route('/template_sample/')
@app.route('/template_sample/<name>')
def from_template(name=None):
    return render_template('template1.html', name=name)

3. 動作確認

実行後、以下のURLで動作確認をしましょう。

  • http://localhost:8888/template_sample/
  • http://localhost:8888/template_sample/Saburo

CSSやJavaScriptなどの静的なファイルを使う

ドキュメントは、以下のURLを参照してください。

  • 静的ファイル — Flask Documentation (2.0.x)

1. ファイルの用意

以下のCSSをstaticディレクトリ以下にindex.cssというファイル名で配置しましょう。

h1 {
    color: red;
}

配置が完了したら、以下のテンプレートをindex.htmlというファイル名で、templatesディレクトリ以下に配置してください。

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Hello</title>
    <link rel="stylesheet" href="{{ url_for('static', filename='index.css') }}">
</head>
<body>
    <h1>Hello, Flask!!</h1>
</body>
</html>

2. main.pyの修正

以下の関数をmain.pyに追加してください。

# Jinjaテンプレートで静的なファイルを参照するサンプル
# http://localhost:8888/index
@app.route('/index')
def static_index():
    return render_template('index.html')

3. 動作確認

$ python main.pyで実行後、以下のURLで動作確認をしましょう。

  • http://localhost:8888/index

CSSにより、装飾がなされていて、Hello, Flask!! の文字が赤くなっていればOKです。

Flaskまとめ

Flaskを使用すると、Webアプリケーションの必要最低限の部分をライブラリにお任せして、プログラマはアプリケーションのロジック開発に専念することが可能となります。 Webアプリケーションフレームワークには、Flask以外にもいくつかあり、それぞれに長所短所があります。 他のライブラリ同様に、自分が作成したいアプリケーションの特徴に応じて使用するライブラリを切り替えると良いでしょう。
