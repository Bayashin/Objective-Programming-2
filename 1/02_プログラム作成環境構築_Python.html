Pythonのインストール

まえおき

  • 画面を見ていてエラーが起こった場合はTAを呼ぶこと

インストール前の確認

  • pyenvがインストールされている人はpyenvのインストールは不要です
  • 確認方法:ターミナルで以下のコマンドを実行してpyenvのバージョンが表示されればOkです
pyenv --version # バージョンが表示されればOK

pyenvのインストール

  1. Pythonのバージョン管理ツールpyenvをhomebrewでインストール
  • 以下のコマンドを実行し，エラーが起こっているように見える場合はTAを呼ぶこと
  • 3行目のcatから始まるコマンドは，最後から３行目のEOS（End Of String）の行までコピーして実行すること
brew install pyenv
brew upgrade pyenv
cat << 'EOS' >> ~/.zshrc
eval "$(pyenv init --path)"
eval "$(pyenv init -)"
eval "$(pyenv virtualenv-init -)"
EOS
# shellをリロードする
zsh -l
  1. pyenv --versionとターミナルに入力してバージョンが表示されることを確認

Pythonインストール　＆　講義用フォルダを作成

端末で以下を実行

pyenv install 3.10.7

他の講義やセミナーと環境が競合しないようにオブジェクト指向プログラミング及び演習２専用のフォルダを用意しましょう

そして，講義用のフォルダ内でのみPython3.10.4が有効になるようにpyenvを設定しましょう

mkdir -p $HOME/work/oop2
cd $HOME/work/oop2
pyenv local 3.10.7
python -V #3.10.7と表示されればOK

（オプション）補足事項

すでにPython3.10.4がpyenvでインストールされていた人は，講義外で作成したプログラムが動かなくなる可能性があります

その場合は，講義専用の仮想環境（virtualenv）を以下のように用意します

cd $HOME/work/oop2
rm .python_version # pyenv localコマンドで作成される隠しフォルダを削除
pyenv virtualenv 3.10.7 oop2
pyenv local oop2 # この行を実行すると端末の先頭に(oop2)と表示される
python -V # 3.10.7と表示されればOK

まとめ

この講義のプログラムを実行するときやプログラムファイルを保存するときは，講義用のフォルダ内で行うこと

  • pythonインタープリタの起動例
cd $HOME/work/oop2
python # quit() もしくは exit()で終了
  • pythonプログラムの起動例
cd $HOME/work/oop2
mkdir lecture01 # サブフォルダの中でも講義環境は有効
cd lecture01 # サブフォルダへ移動
echo "print('Hello World')" > hello.py # pythonプログラムを作成
python hello.py # プログラムファイルを実行
