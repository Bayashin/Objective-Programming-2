CSSフレームワークを使ったUI構築

オブジェクト指向プログラミング及び演習２

デザイン面をフレームワークに任せたUI構築でアプリケーション開発に集中しよう

Webアプリケーションを作成する場合、画面の要素を操作調整するにはCSSの知識が必須です。

CSSで1からアプリケーションのUIを作り上げるのは、かなりの知識と労力を必要とします。 アプリケーション開発における「デザイン」は非機能要件として挙げられることもありますが、場合によっては見た目は内部機能より重要であることが多いです。 開発当初からデザインを意識して開発ができればそれに越したことはありません。 が、最初に言った通り、1からすべてを実装するのはかなりの知識と労力が必要です。

そこで登場するのが、CSSフレームワークです。

CSSフレームワークは、JavaやPythonなどのライブラリと同様に人が作ったデザイン済みのCSSを自分のWebアプリケーションに適用することが可能となります。

CSSフレームワークを用いると、後回しになってしまいがちなアプリケーションのUIデザインをあるものでいったん済ませることができ、機能の開発に集中できます。

ただし、フレームワークを使うことにもデメリットはあります。 主なデメリットは、CSSフレームワーク自体への理解が必要となることです。 このデメリットは、CSSに限らず、どんなフレームワークを使う上ではどんなものでもついてくるデメリットのため、必ず使用するフレームワークの公式ドキュメントはいつでも閲覧できるようにしておきましょう。

代表的なCSSフレームワーク(適当な解説つき)

  • Bootstrap · 世界でもっとも人気のあったフロントエンドフレームワーク
    • もともとTwitter社が作ったOSSのCSSフレームワーク
    • Webアプリケーションへの採用率が非常に高く、インターネットに知見も多いため人気がある
  • Tailwind CSS - Rapidly build modern websites without ever leaving your HTML.
    • コンポーネント志向の強いフロントエンドJavaScriptライブラリ（ReactやVue）と親和性が高い
    • コンポーネント志向を理解していないと、HTMLタグに指定するclass属性の記述量が増えて扱いづらい側面もある
  • Bulma: Free, open source, and modern CSS framework based on Flexbox
    • ちょっと前までは人気だった
  • The most advanced responsive front-end framework in the world. | Foundation
    • Bootstrapと同じ時代から存在する
      • CSS設計の考え方がしっかりと理解できている場合は非常に扱いやすい
  • Pure
    • Yahoo!製のフレームワーク、軽量でシンプル（Yahoo! Japanではない）
  • Semantic UI
    • 結構古くからある
  • Primer CSS
    • GitHub製
  • Skeleton: Responsive CSS Boilerplate
    • 必要最小限しかないので逆に使いやすい
  • new.css
    • classをいっさい汚染しない要素セレクタに対するスタイルの定義のみで構成されたフレームワーク
    • 開発初期である程度のデザインは欲しいけど最終的には違うフレームワークを選定することになりそうなとき、class属性を設定する必要がないので便利に扱えます

他にもさまざまなCSSフレームワークがあります。 どれを使っても良いのですが、複数人での開発を考えた場合には、「知名度の高いもの」「公式ドキュメントが読みやすいもの」「メンバー内で詳しい人がいるもの」などの選定基準で選ぶと良いでしょう。

HTML,CSSに不慣れな方は、メンバーに詳しい人がいる場合は、その人にすぐ聞ければ良いです。 もしその状況でない場合、インターネットでの情報収集のしやすさを基準に考えていただければ良いです。

今回の課題開発に採用するCSSフレームワーク

  • new.css

new.cssは、フレームワークの中でもclassをいっさい設定する必要がないフレームワークのため、CSSフレームワークに対する知識がほぼ不要という最大のメリットがあります。

何はともあれ、CSSの知識よりもHTMLに対する知識をつけて欲しいので、このCSSフレームワークを使用することとしました。
