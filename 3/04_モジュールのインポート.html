自作モジュールの作成

ここではfizzbuzzゲームの出力を指定された回数出力するモジュールの作成を学びます

はじめに，fizzbuzzゲームのプログラムを書くだけならモジュールにする必要はありません

以下のようなプログラム(fizzbuzz_base.py)を実行してみましょう

max_count=10
for i in [x+1 for x in range(max_count)]:
    if i%15 == 0:
        print("FizzBuzz")
    elif i%3 == 0:
        print("Fizz")
    elif i%5 == 0:
        print("Buzz")
    else:
        print(i)

一人でプログラムを書くのであればこのプログラムで何も問題ありませんが，複数人でプログラムを開発することを考えると問題があります

問題１：このファイルを直接変更しないと出力回数が指定できないこと（外部から関数を呼び出すことを想定していない）

問題２：for文の中に直接if文やprint関数が書かれているので，関数化したとしても開発中に呼び出すことがｄけいないこと（呼び出すインターフェースと実行する内容を分離すると複数人で開発できる）

プログラム内容の関数化

ここでは外部からプログラムを呼び出すことを想定して処理を関数化します

関数化の例を見てみましょう

def _logic(num):
    """ fizzbuzzの内部処理をするメソッド
    アンダーバー1個は同じファイル内からのみ呼び出すという宣言
    """
    if num%15 == 0:
        return "FizzBuzz"
    elif num%3 == 0:
        return "Fizz"
    elif num%5 == 0:
        return "Buzz"
    else:
        return str(num)

def fizzbuzz(max_count: int) -> None:
    """ fizzbuzz関数はmax_count回結果を出力します

    parameters
    ----
    max_count: int
        出力回数を指定
    """

    for i in range(max_count):
        print(__logic(i+1))

if __name__ == '__main__':
    val=15
    expected = "FizzBuzz"
    if(__logic(val) == expected):
        print(f"_logic returns '{expected}'")
    else:
        print(f"_logic returns the invalid result({val})")
    val=5
    expected = "Buzz"
    if(_logic(val) == expected):
        print(f"_logic returns '{expected}'")
    else:
        print(f"_logic returns the invalid result({val})")
    val=3
    expected = "Fizz"
    if(_logic(val) == expected):
        print(f"_logic returns '{expected}'")
    else:
        print(f"_logic returns the invalid result({val})")
    val=2
    expected = "2"
    if(_logic(val) == expected):
        print(f"_logic returns '{expected}'")
    else:
        print(f"_logic returns the invalid result({val})")

fizzbuzz関数は外部から呼び出させることを想定して作成しています

外部から呼び出すときに関数の処理内容や引数が分かるようにdocstringが関数宣言の直下に追記されています

また，動作確認に使った定数残っているとこのファイルを外部から読み込んだ時に悪さをするかもしれませんので，動作確認箇所はこのファイルが直接実行されたときだけ実行されます（if __name__ == '__main__':の箇所を参照）

外部から呼び出すことからは外れますが，複数人で開発する時に呼び出す関数（fizzbuzz）と関数内の処理をする関数（_logic）に分けておくと，呼び出す関数（インターフェース）を定義する人と処理内容を書く人で役割分担ができます

次は，fizzbuzz関数を外部のファイルから呼び出す方法を学びます

自作モジュールのインポート

自作のFizzBuzz関数をモジュールとして読み込み（import）して別プログラムで利用します

モジュールを格納するフォルダ名をmymodとして作成してください

mymodフォルダの中にfzフォルダを作成してください．（mymod/fzというフォルダ構成になります）

mymodフォルダ内とfzフォルダ内に__init__.pyという名前のファイルを作成します．

  • ディレクトリ構造は以下のようになります
main.py
mymod
    ❙ __init__.py
    ┗ fz
        ┗ fizzbuzz.py

__init__.pyの内容

from mymod.fz.fizzubzz import fizzbuz
#from mymod.fz.fizzubzz import *

外部から呼び出したい関数をすべて並べておく

fromの後ろにはフォルダ名とファイル名をドットで区切って書く

importの後ろには呼び出す関数名を書く

main.pyの内容

import mymod

mymod.fizzbuzz(10) # 関数を呼び出し

まとめ

自作関数をモジュールとして読み込む方法を紹介した．

Pythonモジュールのimportは次の順番で読み込むことが推奨されている．

  1. Pythonネイティブもジュール(csv, xml.etree.ElementTreeなど)
  2. pipなどでインストールした外部モジュール(numpyなど)
  3. 自作モジュール

自作関数の中でも特に外部から呼び出されることが想定される関数にはdocstringを書くようにしましょう
