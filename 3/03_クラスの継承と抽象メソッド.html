クラスの継承

既存のクラスの変数やメソッドや引き継いだ子クラスを定義します．

はじめに，lecture03_hero.pyをmy_moduleフォルダの中に移動します（my_moduleフォルダは作成してください）

次に，lecture03_magician.pyをMoodleからダウンロードしてmy_moduleと同じ階層（lecture03_hero.pyの1つ上のフォルダ）に置きます．

from my_module.lecture03_hero import Hero
# 別ファイルからHeroクラスを読み込み
# from A import Bの構文については，次の資料で詳細を解説

# Heroクラスを継承したMagicianクラスを宣言
class Magician(Hero): 

    __DEFAULT_MP = 100

    # コンストラクタ
    # 親クラスのコンストラクタをsuper().__init__()で呼び出すことができる
    # 親クラスのprivate変数にアクセスする方法１としてDEFAULT_HPのゲッター関数を呼び出している
    def __init__(self, name=None, hp=Hero.getDefaultHP(), mp=__DEFAULT_MP):
        super().__init__(name, hp)
        self.mp = mp

    # 特殊な関数の宣言(上書き)
    def __str__(self):
        return f'"名前": "{self.name}", "HP": {self.hp + self.buf_status["hp"]}, "MP": {self.mp}'

    # 親クラスのprivate変数へのアクセス
    def printDefaultHP(self):
        # 親クラスのprivate変数にアクセスする方法2
        print(f"親クラスのプライベート変数にアクセス: DEFAULT_HP={Hero._Hero__DEFAULT_HP}")

if __name__ == '__main__':
    hero1 = Hero('愛知太郎') # Heroクラスのインスタンスを生成
    print(f'hero1={str(hero1)}') # 文字列に変換
    hero2 = Magician(name='愛知花子', hp=200, mp=200) # 引数名を指定してコンストラクタを呼び出し
    print(f'hero2={str(hero2)}') # 文字列に変換
    print(type(hero1)) # <class 'my_module.lecture03_hero.Hero'>
    print(type(hero2)) # <class '__main__.Magician'>
    if type(hero1) is Hero:
        print('hero1はHero')
    if type(hero2) is not Hero:
        print('hero2とはHeroではない')
    if isinstance(hero1, Hero):
        print('hero1はHeroクラスのインスタンス')
    if isinstance(hero2, Hero):
        print('hero2もHeroクラスのインスタンス')

ポイント

以下のチェック項目について該当箇所と実行結果を探してみよう

  1. Heroクラスは別のファイルに宣言されているのでFrom A import Bの構文で使えるように呼び出す（詳細は次の資料にて）

  2. クラスの継承はclass Magician(Hero):のように括弧内に親クラスの名前を書いて宣言する

  3. 親クラスのメソッドと同じ名前のメソッドを宣言できる（名前空間が異なるので親クラスのメソッドが上書きされることはない）

    • 上書きされるわけではないが，小クラスのメソッドが呼び出されるようになる

    • 親クラスのメソッドは上書きされていないので，super().__init__(name, hp)のように呼び出すことができる

  4. 親クラスで宣言された変数nameやhpを参照できる

    • この箇所を参照 return f'"名前": "{self.name}", "HP": {self.hp + self.buf_status["hp"]}, "MP": {self.mp}'

  5. 親クラスのプライベート変数やプライベート関数は基本アクセスできない

    • アクセス方法１：GetterやSetterメソッドを宣言しておく(getDefaultHPメソッドがGetterに当たる)

    • アクセス方法２：Hero._Hero__DEFAULT_HPのように特殊な変数名に置き換わっているだけなので読み書き可能

抽象メソッド

Javaのインターフェースに相当する抽象メソッドの宣言と利用方法について説明します．

はじめに，MoodleよりダウンロードしたIMagicAttack.pyをmy_moduleフォルダの中にコピーしてください．

import abc
from lecture03_hero import Hero

class IMagicAttack(metaclass=abc.ABCMeta):
    @abc.abstractmethod
    def damage(self, enemy : Hero) -> None:
        """
        魔法攻撃のダメージを敵に与える
        ダメージを与えた後にHPが0以下になったenemyを削除する

        Parameters
        ----------
        enemy : Hero
            ダメージを与えるHero
        """
        raise NotImplementedError()

次に，lecture03_magician2.pyをMoodleからダウンロードしてmy_moduleと同じ階層（lecture03_hero.pyとIMagicAttack.pyの1つ上のフォルダ）に置きます．

from my_module.lecture03_hero import Hero
from my_module.IMagicAttack import IMagicAttack

# Heroクラスを継承したMagicianクラスを宣言
class Magician2(Hero, IMagicAttack): 

    __DEFAULT_MP = 100

    # コンストラクタ
    def __init__(self, name=None, hp=Hero.getDefaultHP(), mp=__DEFAULT_MP):
        super().__init__(name, hp)
        self.mp = mp

    # 特殊な関数の宣言(上書き)
    def __str__(self):
        return f'"名前": "{self.name}", "HP": {self.getHP()}, "MP": {self.mp}'

if __name__ == '__main__':
    hero3 = Magician2(name='愛知さぶろう', hp=10)
    hero3.damage(hero3) # 実装したのでエラーではなくhpが減り離脱する

エラーを解消する（プログラムを完成させる）ために，抽象メソッドを実装する（Magician2クラス内に宣言してください）

    # 抽象クラスのメソッドを実装する
    def damage(self, enemy: Hero) -> None:
        print(f"{self.name}は{enemy.name}に100のダメージ")
        enemy.hp -= 100
        # bufを含めhpが0以下の場合は離脱させる
        if(enemy.getHP() <= 0):
            del(enemy)

実行結果

ポイント

  • 抽象メソッド(abc)を使うとJavaのインターフェースのように関数名だけを宣言しておいて，実装は継承したクラス内に任せることができる

  • クラス宣言時に多重継承（複数の親クラスを持つこと）ができる（Javaではクラスの多重継承はできないがインターフェースの多重継承はできる）
